# dclone-relay

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/new/template?template=https://github.com/DriftingNarwhal/dclone-relay)

A standalone relay server for [DClone](https://github.com/DriftingNarwhal/DClone), a P2P encrypted messaging app built with Rust and libp2p. When two DClone users are both behind NAT, direct connections are impossible without help. This relay server bridges that gap by acting as a circuit relay v2 node: DClone clients first attempt a direct dial, then try DCUtR hole-punching through the relay, and if that still fails they fall back to a persistent relay circuit through this server. The relay also runs a rendezvous server so peers can find each other without a central coordinator.

---

## Deploy to Railway

> **One-click deploy** — click the button above, set one environment variable, and you're done.

### Step-by-step setup

**Step 1 — Click the Deploy button** above (or go to [railway.app](https://railway.app) → New Project → Deploy from GitHub repo → select this repo).

**Step 2 — Railway detects the Dockerfile** and starts building automatically. The build takes a few minutes on first run.

**Step 3 — Set your keypair variable.** Once the build finishes, go to your service → **Variables** tab and add:

| Key | Value |
|-----|-------|
| `RELAY_KEYPAIR` | The base64 value from your `keypair.env` file |

This value comes from the **Generate Relay Setup** button inside the DClone app. Open the downloaded ZIP, find `keypair.env`, and copy the value that appears after `RELAY_KEYPAIR=`.

**Step 4 — Expose port 4001 as a TCP service.** Railway will redeploy automatically after you add the variable. Once the health check passes, go to **Settings → Networking** and add a **TCP Proxy** on port `4001` to get a public hostname and port for libp2p traffic.

**Step 5 — Add the relay in DClone.** In the DClone app, use **Verify and Add Relay** with the Railway-assigned TCP hostname and port `4001`.

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `RELAY_KEYPAIR` | **Yes** | Base64 protobuf-encoded Ed25519 keypair. Generated by the DClone app's **Generate Relay Setup** button. |
| `PORT` | No (Railway sets it) | HTTP health-check port. Railway injects this automatically — do not set it manually. |
| `RUST_LOG` | No | Log level. Default: `info,libp2p_relay=debug` |

---

## Ports

| Port | Protocol | Purpose |
|---|---|---|
| 4001 | TCP | libp2p relay — **must be exposed via Railway TCP proxy** |
| 4001 | UDP | QUIC transport (best-effort — Railway shared networking may not support UDP) |
| `$PORT` | TCP / HTTP | Health-check endpoint (`/health`) — Railway routes HTTPS traffic here |

> **Note on QUIC:** If Railway cannot bind the UDP port, the relay logs a warning and continues operating over TCP only. DClone clients will fall back to TCP automatically.

## Health-Check Endpoint

```
GET https://<your-railway-domain>/health
→ {"peer_id":"12D3KooW..."}
```

This peer ID must match the one shown in the DClone app after keypair generation. If they don't match, the app will reject the relay.

Railway uses this endpoint (`/health`) to determine whether the service is healthy before routing traffic to it. The `healthcheckTimeout` in `railway.toml` is set to 300 seconds to allow time for the Rust build and startup.

## Security

`RELAY_KEYPAIR` identifies your relay on the network. If it leaks, anyone can impersonate your relay node.

- **Never commit it to git.** The `.gitignore` in this repo excludes all `*.env` files including `keypair.env`.
- Set it only via the Railway **Variables** tab (or your deployment platform's secret store).
- If your keypair is compromised, generate a new one in the DClone app and redeploy.

## Self-Hosted / Docker Alternative

If you prefer to run the relay yourself instead of using Railway:

```bash
docker run -d \
  --name dclone-relay \
  --restart unless-stopped \
  -e RELAY_KEYPAIR=<your_base64_keypair> \
  -p 4001:4001/tcp \
  -p 4001:4001/udp \
  -p 4002:4002/tcp \
  ghcr.io/DriftingNarwhal/dclone-relay
```

Check health:

```bash
curl http://localhost:4002/health
```

The relay is healthy when it returns `{"peer_id":"12D3KooW..."}`.

## Building from Source

```bash
git clone https://github.com/DriftingNarwhal/dclone-relay
cd dclone-relay
cargo build --release
RELAY_KEYPAIR=<your_key> ./target/release/dclone-relay
```
